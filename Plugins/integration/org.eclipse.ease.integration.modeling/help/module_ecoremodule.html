Module EcoreModule

This offer to user means to handle EMF models (Create EObject, Select element, Handle resources etc..). This module need to be initialized with an nsURI referencing a metamodel usin {@link EcoreModule#initEPackage(String)}.Once this module has been initialized all creation method of the factory is injected in the script.

Function Overview

eInstanceOf Return if the current instance is a instance of an EClass define by its name.
getSelection Returns the currently selected model element in the current editor if it is an instance of the named meta-class (or a sub-class).
initEPackage Initialized the module with the correct metamodèle. If this method is not called the module will at runtime ask with metamodel shall be used.
createResource Create a new resource to hold model elements
createURI Create a new URI. This URI is use to locate a resource.
getFactory Get the factory of selected meta model.
getEPackage Get the {@link EPackage} of the selected meta model
addErrorMarker Add an error marker on a EObject
addInfoMarker Add an Information marker on a EObject
addWarningMarker Add a Warning marker on a EObject
save Save: The current editor if no eObject is passed in argument The resource containing the eObject passed in argument
getShell
getUsages Return all object referencing this EObject. The return value is a collection of Array of size 2. Result[0] = EStructual feature linking the two object Result[1] = The referencing object
runOperation Run an operation in the current editor's command stack This is really help ful to manipulate a model using transaction
selectFromList Display a dialog which ask the user to select between a list of Object
ePrint Print an {@link EObject} using label providers
